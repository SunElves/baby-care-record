# 局域网访问配置说明

## 当前配置状态

✅ **开发服务器已启动并配置完成**
- 本地访问：`http://localhost:3000/`
- 局域网访问：`http://192.168.10.235:3000/`
- 端口：3000
- 防火墙状态：已禁用（不会阻止局域网访问）

## 如何使用局域网访问

### 1. 确保设备在同一网络
- 确保您的手机、iPad或其他设备与开发电脑连接到同一个WiFi网络
- 不要使用不同的WiFi或移动数据网络

### 2. 在其他设备上访问
在局域网内的其他设备浏览器中输入：
```
http://192.168.10.235:3000/
```

### 3. 如果IP地址发生变化
每次重启网络或连接不同WiFi后，IP地址可能会改变。重新启动开发服务器时会显示最新的网络地址。

## 故障排除

### 无法访问的可能原因：

1. **设备不在同一网络**
   - 检查所有设备是否连接到同一个WiFi
   - 确认没有使用VPN或代理

2. **IP地址已变化**
   - 查看开发服务器启动时显示的最新Network地址
   - 或在终端运行：`ifconfig | grep "inet " | grep -v 127.0.0.1`

3. **防火墙阻止**
   - 当前macOS防火墙已禁用，无需额外配置
   - 如果后续启用了防火墙，需要允许Node.js通过

4. **路由器设置**
   - 某些路由器可能禁用了设备间通信
   - 检查路由器的AP隔离功能是否开启

## 开发服务器配置

配置文件位置：`client/vite.config.js`

```javascript
server: {
  port: 3000,           // 访问端口
  host: '0.0.0.0',      // 允许局域网访问
  strictPort: true,     // 端口被占用时直接失败
  open: false           // 不自动打开浏览器
}
```

## 重启开发服务器

如果需要重启服务器：
1. 停止当前服务器（Ctrl+C）
2. 运行：`cd client && npm run dev`
3. 查看输出的Network地址

## 安全提示

⚠️ **重要提醒**：
- 此配置仅用于开发环境
- 不要在生产环境使用此配置
- 局域网访问意味着同一网络内的设备都可以访问
- 开发完成后请确保使用生产环境部署方案
